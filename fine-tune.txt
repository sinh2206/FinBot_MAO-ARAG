HƯỚNG DẪN CHẠY CONVERT RAW DATA VÀ BUILD INDEX
===============================================

Mục tiêu
--------

Pipeline dữ liệu hiện tại có 2 bước chính:

1. Chuyển file gốc trong `data/raw_data/` thành file `.txt` UTF-8 trong `data/processed_data/`.
   Script sử dụng: `scripts/convert_raw_data.py`.

2. Build RAG index từ `data/processed_data/` để tạo chunks, embeddings, FAISS/BM25 index.
   Script sử dụng: `scripts/build_index.py`.

Không nên đưa trực tiếp file PDF/DOC gốc vào fine-tune. Nên convert sang TXT trước, kiểm tra OCR, sau đó mới build index và tạo bộ QA/train data.

Chạy nhanh trong dự án này:

```bash
python scripts/convert_raw_data.py \
  --input_dir data/raw_data \
  --output_dir data/processed_data \
  --metadata_dir data/metadata \
  --ocr auto

python scripts/build_index.py \
  --data_dir data/processed_data \
  --chunk_dir data/chunks \
  --index_dir data/index \
  --embedding_dir data/embeddings \
  --metadata_dir data/metadata \
  --local_files_only
```


1. Cài môi trường
-----------------

Chạy trong môi trường Python của dự án:

```bash
pip install -r requirements.txt
```

Nếu cần OCR file PDF scan, máy phải có `tesseract` và gói ngôn ngữ tiếng Việt.

Trên Ubuntu/Debian:

```bash
sudo apt-get update
sudo apt-get install -y tesseract-ocr tesseract-ocr-vie tesseract-ocr-eng antiword
```

Kiểm tra Tesseract:

```bash
tesseract --list-langs
```

Cần thấy có:

```text
vie
eng
```


2. Chuẩn bị dữ liệu gốc
-----------------------

Đặt file gốc vào:

```text
data/raw_data/
```

Ví dụ:

```text
data/raw_data/BFC-Q3.pdf
data/raw_data/BKG-Q4.pdf
data/raw_data/BMI-Q4.pdf
```

`convert_raw_data.py` hỗ trợ:

```text
.pdf
.docx
.doc
.txt
.md
.markdown
.html
.htm
.csv
```


3. Convert raw_data sang processed_data
---------------------------------------

Lệnh chạy mặc định:

```bash
python scripts/convert_raw_data.py \
  --raw_dir data/raw_data \
  --processed_dir data/processed_data \
  --metadata_dir data/metadata \
  --ocr auto
```

Ý nghĩa:

`--raw_dir`: thư mục chứa file gốc.

`--processed_dir`: thư mục nhận file `.txt` đã xử lý.

`--metadata_dir`: thư mục ghi summary quá trình convert.

`--ocr auto`: nếu PDF trích text quá ít thì tự động OCR.

Nếu muốn ghi đè toàn bộ file TXT đã convert:

```bash
python scripts/convert_raw_data.py \
  --raw_dir data/raw_data \
  --processed_dir data/processed_data \
  --metadata_dir data/metadata \
  --ocr auto \
  --overwrite
```

Nếu PDF scan bị OCR chậm, có thể giảm DPI:

```bash
python scripts/convert_raw_data.py \
  --raw_dir data/raw_data \
  --processed_dir data/processed_data \
  --metadata_dir data/metadata \
  --ocr auto \
  --ocr_dpi 120
```

Nếu chỉ muốn trích text có sẵn trong PDF, không OCR:

```bash
python scripts/convert_raw_data.py \
  --raw_dir data/raw_data \
  --processed_dir data/processed_data \
  --metadata_dir data/metadata \
  --ocr never
```


4. Kiểm tra kết quả convert
---------------------------

Kiểm tra file TXT đã sinh:

```bash
find data/processed_data -maxdepth 1 -type f -name "*.txt" -printf "%f %s bytes\n" | sort
```

Kiểm tra summary:

```bash
python - <<'PY'
import json
from pathlib import Path

summary = json.loads(Path("data/metadata/raw_data_conversion_summary.json").read_text(encoding="utf-8"))
print("total_files:", summary["total_files"])
print("converted:", summary["converted"])
print("skipped:", summary["skipped"])
print("failed:", summary["failed"])
print("ocr_mode:", summary["ocr_mode"])
print("ocr_lang:", summary["ocr_lang"])
print("ocr_dpi:", summary["ocr_dpi"])
PY
```

Nếu `failed > 0`, xem chi tiết file lỗi:

```bash
python - <<'PY'
import json
from pathlib import Path

summary = json.loads(Path("data/metadata/raw_data_conversion_summary.json").read_text(encoding="utf-8"))
for item in summary["files"]:
    if item["status"] == "failed":
        print(item["source"], "=>", item.get("error"))
PY
```


5. Kiểm tra lỗi OCR tiếng Việt
------------------------------

Script convert đã có bước sửa một số lỗi OCR chắc chắn, ví dụ:

```text
Ching khoén / Chtmg khoan / CHUNG KHOAN -> Chứng khoán
Thanh phé Ho Chi Minh / Thanh ph6 HO Chi Minh -> Thành phố Hồ Chí Minh
S6 Giao dich / So Giao dich / S& Giao dịch -> Sở Giao dịch
```

Kiểm tra nhanh pattern lỗi còn sót:

```bash
rg -n "Ching kho|Chimg kho|Chtmg kho|CHUNG KHOAN|Thanh phé|Thanh ph6|S6 Giao dich|S& Giao" data/processed_data
```

Nếu không có output là tốt.

Kiểm tra các cụm đã được sửa:

```bash
rg -n "Sở Giao dịch|Chứng khoán|Thành phố Hồ Chí Minh|TP\\. Hồ Chí Minh" data/processed_data | head -50
```


6. Build RAG index từ processed_data
------------------------------------

Sau khi TXT đã ổn, build index:

```bash
python scripts/build_index.py \
  --data_dir data/processed_data \
  --chunk_dir data/chunks \
  --index_dir data/index \
  --embedding_dir data/embeddings \
  --metadata_dir data/metadata \
  --local_files_only
```

Nếu chưa có model embedding trong local cache/models, bỏ `--local_files_only` để tải từ Hugging Face:

```bash
python scripts/build_index.py \
  --data_dir data/processed_data \
  --chunk_dir data/chunks \
  --index_dir data/index \
  --embedding_dir data/embeddings \
  --metadata_dir data/metadata
```

Nếu muốn chỉnh chunk:

```bash
python scripts/build_index.py \
  --data_dir data/processed_data \
  --chunk_size 384 \
  --chunk_overlap_ratio 0.2 \
  --dense_weight 0.7 \
  --sparse_weight 0.3 \
  --local_files_only
```

9. Liên hệ với fine-tune LFM2-1.2B-RAG
--------------------------------------

`data/processed_data/` là corpus text đã xử lý.

`data/chunks/chunks.json` là nguồn context đã chunk để tạo câu hỏi/câu trả lời.

`data/index/` là index phục vụ retrieval.

Để fine-tune LFM2-1.2B-RAG, không fine-tune trực tiếp từ PDF. Nên tạo bộ QA từ:

1. `data/processed_data/*.txt`
2. `data/chunks/chunks.json`
3. Câu hỏi/câu trả lời mẫu do người kiểm tra lại

Sau đó chia thành:

```text
train: dùng để fine-tune
validation: chọn checkpoint, chỉnh prompt/hyperparameter
test: chỉ đánh giá cuối, không dùng trong lúc train
```

Quy trình đúng:

```text
raw_data -> convert_raw_data.py -> processed_data -> build_index.py -> chunks/index -> tạo QA -> split train/validation/test -> fine-tune LFM2-1.2B-RAG
```
